# Load Balancer

## What is a Load Balancer?

A Load Balancer is a service that automatically distributes incoming network traffic across multiple servers or EC2 instances to improve performance, availability, and reliability.

## Why do we need a Load Balancer?

* Distributes traffic evenly across multiple servers
* Prevents a single server from becoming overloaded
* Improves application performance
* Increases availability
* Reduces downtime

## Features

* Traffic distribution
* High availability
* Fault tolerance
* Health checks
* SSL/TLS termination
* Supports horizontal scaling

## Advantages

* Better performance
* Improved reliability
* Automatic failover
* High availability
* Better user experience

## Real Life Example

An e-commerce website receives thousands of customer requests during a festival sale. A Load Balancer distributes these requests across multiple EC2 instances so that no single server becomes overloaded.

## Key Points

* Distributes incoming traffic
* Improves performance and availability
* Prevents server overload
* Supports High Availability
* Used with Auto Scaling Group (ASG)
