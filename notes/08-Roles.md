# AWS IAM ROLES
## What is an IAM role??
- An IAM role is an AWS identity that provides temporary permissions to access AWS resources without using long-term credentials.

## Why we use IAM Roles??
- Provides temporary security credentials.
- Eliminate the need to share access key.
- improves security.
- Easy to manage permissions.

## Common Use:-
- EC2 instances accessing S3 buckets.
- Lambda functions accessing AWS services.
- Cross-account access.
- Federated user access.

## Benefits:-
- Enhanced security.
- Temporary credentials.
- Easy permission management.
- Supports least privilage principle.
